## 1. Description

This file presents the structural model of system behavior under regulatory conditions,
derived from reproducibility, structural filtering, and validation.

- the core causal mechanism

## 2. Core Mechanism

> self-reporting  → information control  → selective disclosure  → verification limitation  → detection gap  

Outcome: constrained detection capacity  

Validation status: empirically confirmed  

Basis:

- core mechanism reproduced across runs  
- structurally stable after filtering  
- no logical contradictions  
- behavioral elements excluded from core  

Core exclusions:

- incentive-driven behavior  
- interpretative behavior  
→ treated as secondary or conditional  

Implication:

→ detection limitations are structural, not purely enforcement-driven  
→ increasing enforcement alone is unlikely to resolve the constraint  

---

## 2.1 Mechanism Explanation

**self-reporting → information control**  
• cause: reporting architecture places data generation under control of regulated entities  
• factors: F7 — Conflict of Interest (information control)  
• type: structural  

**information control → selective disclosure**  
• cause: control over data creates conditions for variation in disclosure scope  
• factors: F7 — Conflict of Interest (information control)  
• type: structural-operational  

**selective disclosure → verification limitation**  
• cause: incomplete or selective data constrains independent verification  
• factors: F5 — Enforcement Gap (verification limitation)  
• type: structural-operational  

**verification limitation → detection gap**  
• cause: limited verification reduces probability of detecting violations  
• factors: F5 — Enforcement Gap (verification limitation)  
• type: structural  

---

## 6. Feedback Structure

detection gap  
→ reduced risk of detection  
→ selective disclosure  
→ verification limitation  
→ detection gap  

Explanation  

**detection gap → reduced risk of detection**  
• cause: low probability of detection reduces expected cost of non-compliance  
• factors: F5  

**reduced risk → selective disclosure**  
• cause: lower risk enables continued selective reporting  
• factors: F5, F7  

**selective disclosure → verification limitation**  
• cause: incomplete information constrains verification  
• factors: F5  

**verification limitation → detection gap**  
• cause: limited verification sustains low detection probability  
• factors: F5  

Validation status: partially confirmed  

Loop Properties  

Type: STRUCTURAL-OPERATIONAL  
Strength: medium  
Stability: conditional  

**Note:**  
The feedback loop is not required for the core mechanism to operate,  
but can amplify it under certain conditions.

## 7. Interpretation

The system operates through a structure where:

- information is generated and controlled by regulated actors  
- disclosure scope is variable  
- verification depends on available data  

→ outcomes are driven by information control and verification limitations
